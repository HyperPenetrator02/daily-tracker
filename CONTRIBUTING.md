# Contributing to StatMaxer RPG OS

Thanks for your interest in contributing! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/daily-tracker.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test thoroughly
6. Commit with clear messages
7. Push to your fork
8. Open a Pull Request

## Development Setup

```bash
# Install dependencies
npm install

# Run local development server
npm run dev

# Test PWA features
# Open http://localhost:8080 in your browser

# Build Android APK (requires Android Studio)
npm run sync
npm run android
```

## Code Style

- Use clear, descriptive variable names
- Add comments for complex logic
- Follow existing code structure
- Keep functions focused and small
- Test on multiple devices/browsers

## Commit Messages

Write clear commit messages:
- Use present tense ("Add feature" not "Added feature")
- Be specific about what changed
- Reference issues when applicable

Examples:
- `Add notification sound customization`
- `Fix streak calculation bug`
- `Update mobile layout for tablets`

## Pull Request Process

1. Update README.md if needed
2. Update CHANGELOG.md with your changes
3. Ensure all features work on mobile and desktop
4. Test the PWA installation
5. Describe your changes clearly in the PR

## Reporting Bugs

Use GitHub Issues and include:
- Clear description of the bug
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable
- Device/browser information

## Feature Requests

We welcome feature suggestions! Please:
- Check existing issues first
- Describe the feature clearly
- Explain the use case
- Consider implementation complexity

## Questions?

Feel free to open a Discussion on GitHub for:
- General questions
- Implementation ideas
- Design feedback
- Collaboration opportunities
